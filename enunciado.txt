El objetivo de esta práctica es construir una aplicación web utilizando React que consuma datos reales de la SWAPI (Star Wars API). La aplicación debe listar personajes en tarjetas, gestionar los estados de espera y error, y permitir cargar más resultados mediante paginación.



Instrucciones generales
Crear un repositorio de GitHub para almacenar todo el código de las practicas. https://github.com
Usar Typescript como lenguaje
Crear una carpeta de practica-1 donde dentro contendrá un archivo por cada ejercicio
El desarrollo es individual
NO esta permitido el uso de IA generativa para resolver la actividad




1. Requisitos Técnicos
1.Lógica de Componentes:

Componente Principal (App): Debe contener la lógica de llamada a la API.
Componente CharacterList: Recibirá los datos como props y se encargará de renderizar la lista.
Componente CharacterCard: Recibirá un objeto personaje como prop y pintará su información individual.
Componente Loader y Error: Componentes simples para mostrar estados de carga y fallos.
2.Gestión de Estado y Hooks:

Implementar useEffect para realizar la llamada a https://swapi.dev/api/people/ al montar el componente.
Implementar useState para gestionar al menos tres estados:
characters: Array con la lista de personajes.
loading: Booleano para mostrar un spinner o texto de "Cargando...".
error: String o null para mostrar mensajes si la petición falla.
3.Paginación:

La API devuelve una URL next en el JSON.
Implementar un botón "Siguiente Página" que, al ser pulsado, realice una nueva petición a esa URL y acumule los nuevos personajes en el estado existente (no reemplazarlos).
